Mission

We exist to make every customer interaction matter

Area was built by a team of researchers, engineers, and operators who spent years inside the world's best AI labs and customer support operations. We saw how hard it was to deploy AI that actually works in production — and built the platform we wished existed.

Today, Area powers support operations for companies from high-growth startups to Fortune 500 enterprises — handling millions of conversations per day with the accuracy and empathy customers deserve.

Safety above all
Every model we ship is evaluated for safety, reliability, and edge-case behavior. We invest heavily in alignment research to ensure Area remains trustworthy as it scales.
Transparency by default
We publish our research, share our evals, and communicate clearly when things go wrong. Building trust requires being honest about limitations as well as capabilities.
Customer-first always
Our models are measured by real customer outcomes — CSAT, resolution rate, time to resolve. Technology only matters when it makes a measurable difference in people's lives.

Getting started

Quickstart

This guide will have you making your first API call to Area in under five minutes. You will need an API key — you can create one in the Console.

Note: Area API keys are project-scoped. Keep them secret and never commit them to version control.

Step-by-step setup

1
Create an account

Sign up at area.com and create a new project from your dashboard. Each project has its own API keys and usage limits.

2
Generate an API key

Go to Console → API Keys → Create new key. Name it clearly (e.g. "dev-local") and copy it immediately — it will not be shown again.

3
Install the SDK
npm install @area/sdk # or pip install area-sdk
4
Make your first call
import Area from '@area/sdk'; const area = new Area({ apiKey: process.env.AREA_API_KEY }); const message = await area.messages.create({ model: 'area-3-sonnet', max_tokens: 1024, messages: [ { role: 'user', content: 'How can I track my order?' } ] }); console.log(message.content[0].text);

Response format

Every API response includes a content array with one or more blocks. For text responses, you will typically access message.content[0].text.

{ "id": "msg_01xabc...", "type": "message", "role": "assistant", "model": "area-3-sonnet", "content": [ { "type": "text", "text": "To track your order, please visit the Orders section..." } ], "usage": { "input_tokens": 12, "output_tokens": 38 } }
